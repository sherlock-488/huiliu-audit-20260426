### V2.4 (2026-03-24 下午)
**新增 1 个过滤规则（规则 17）+ 修复退款方案格式兼容性**

#### 新增规则

**17. 小象单商品退款上下文检查** ⭐
- **检查对象**: `ResponseSolution` + `context`
- **规则**: 如果使用"小象单商品退款"方案，对话历史中必须包含"图片"或"选择"
- **原因**: 用户需要选择商品并提供商品图片
- **匹配模式**:
  - 方案匹配: `小象-?单商品退款-\d+%`（支持两种格式）
  - 上下文要求: 必须包含"图片"或"选择"
- **示例**:
  - ❌ 方案是"小象单商品退款-50%"，但上下文中没有"图片"或"选择"
  - ❌ 方案是"小象-单商品退款-100%"，但用户只说"就那个"
  - ✅ 方案是"小象单商品退款-50%"，上下文有"请您选择需要退款的商品"
  - ✅ 方案是"小象-单商品退款-100%"，上下文有"请您上传商品图片"
  - ✅ 方案是"小象整单申请退款"（不是单商品退款，无需检查）

#### 修复问题

**修复退款方案格式兼容性**
- **问题**: `check_illegal_solution` 函数只匹配 `小象单商品退款-xx%`，不匹配 `小象-单商品退款-xx%`
- **修复**: 正则表达式改为 `r'^小象-?单商品退款-(\d+)%$'`，支持两种格式
- **影响**: 修复前，`小象-单商品退款-xx%` 会被误判为"非法工具/方案"

## 规则实现细节

### 规则 17: 小象单商品退款上下文检查

```python
def check_xiaoxiang_refund_context(response_solution, context):
    """
    17. 检查小象单商品退款是否有必要的上下文（图片或选择）

    规则: 如果使用"小象单商品退款"方案，对话历史中必须包含"图片"或"选择"
    原因: 用户需要选择商品并提供商品图片
    """
    # 检查是否是小象单商品退款方案（支持两种格式）
    pattern = r'小象-?单商品退款-\d+%'
    is_xiaoxiang_refund = bool(re.search(pattern, response_solution))

    if not is_xiaoxiang_refund:
        return False  # 不是小象单商品退款，不需要检查

    # 检查必要条件：必须包含 "图片" 或 "选择"
    has_required_context = ('图片' in context) or ('选择' in context)

    if not has_required_context:
        return True  # 缺少必要的上下文，过滤

    return False  # 有必要的上下文，通过
```

**业务逻辑**:
1. 小象单商品退款需要用户明确选择哪个商品
2. 用户需要提供商品图片作为退款依据
3. 如果对话历史中没有"图片"或"选择"关键词，说明流程不完整

**支持的方案格式**:
- `小象单商品退款-50%` ✅
- `小象-单商品退款-100%` ✅
- `小象单商品退款-80%` ✅

**不检查的方案**:
- `小象整单申请退款` （整单退款，无需选择商品）
- `操作赔付-优惠券-5元` （赔付方案，不是退款）
- `加急调度` （其他工具）
