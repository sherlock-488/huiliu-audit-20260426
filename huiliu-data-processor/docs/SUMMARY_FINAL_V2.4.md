## 🔥 核心技术亮点

### 1. 两阶段处理架构
```
阶段 1: 数据转换
  - 解析 TSV/JSON 输入
  - 提取 user_reply（从对话历史）
  - 输出 JSONL（带 user_reply 字段）

阶段 2: 数据过滤
  - 应用 17 个过滤规则
  - 智能参数传递
  - 输出干净的训练数据
```

### 2. 智能参数传递
每个规则使用正确的检查对象：
- `user_reply`: 用户认可度、用户负面反馈、转人工关键词
- `Response`: 过度承诺、口语词、长度、重复、退货、业务名称、打电话、升级转接、退款越权
- `ResponseSolution`: 非法工具、信号逻辑、赔付冲突、小象退款上下文
- `input_prompt`: 信号提取、赔付权限、退款豁免
- `context`: 对话上下文（客服声称真人、小象退款上下文）

### 3. 正则表达式优化
- **负向预查**: `(?!申请)(?!审核)` 排除过程态表述
- **可选匹配**: `.{0,6}` 处理中间插入词
- **多格式支持**: `小象-?单商品退款` 兼容两种格式
- **多条件组合**: 提高匹配准确率

### 4. 业务逻辑处理
- **豁免条件**: 退款越权规则在"已全部退款"时豁免
- **信号匹配**: 加急调度、取消订单、催促退款的信号一致性验证
- **上下文检查**: 小象单商品退款需要"图片"或"选择"

### 5. Bug 修复
**问题**: `check_illegal_solution` 只匹配 `小象单商品退款-xx%`，不匹配 `小象-单商品退款-xx%`
**修复**: 正则表达式改为 `r'^小象-?单商品退款-(\d+)%$'`
**影响**: 修复前会误判为"非法工具/方案"

## 🎓 核心规则详解

### 规则 17: 小象单商品退款上下文检查（最新）

**业务背景**:
- 小象超市的单商品退款需要用户明确选择哪个商品
- 用户需要提供商品图片作为退款依据
- 如果对话历史中没有"图片"或"选择"，说明流程不完整

**实现逻辑**:
```python
def check_xiaoxiang_refund_context(response_solution, context):
    # 1. 检查是否是小象单商品退款方案（支持两种格式）
    pattern = r'小象-?单商品退款-\d+%'
    is_xiaoxiang_refund = bool(re.search(pattern, response_solution))

    if not is_xiaoxiang_refund:
        return False  # 不是小象单商品退款，不需要检查

    # 2. 检查必要条件：必须包含 "图片" 或 "选择"
    has_required_context = ('图片' in context) or ('选择' in context)

    if not has_required_context:
        return True  # 缺少必要的上下文，过滤

    return False  # 有必要的上下文，通过
```

**匹配示例**:
- ✅ `小象单商品退款-50%` + context 有"选择"
- ✅ `小象-单商品退款-100%` + context 有"图片"
- ✅ `小象单商品退款-80%` + context 有"选择"和"图片"
- ❌ `小象单商品退款-50%` + context 无"图片"或"选择"
- ✅ `小象整单申请退款` （不是单商品退款，无需检查）
