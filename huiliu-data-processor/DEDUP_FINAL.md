# 最终训练数据重复度筛选

## 概述

`dedup_final.py` 是对已生成的最终训练数据进行重复度检测和去重的脚本。

**特点**：
- 支持完全相同去重和相似度去重
- 提供多种相似度计算方法
- 灵活的字段选择（支持嵌套字段）
- 详细的统计信息

## 使用场景

### 场景1：完全相同的回复去重

如果训练数据中存在完全相同的回复（可能来自不同的用户问题），可以按 Response 字段去重：

```bash
python3 dedup_final.py training_data.jsonl output.jsonl --key-field Response
```

### 场景2：基于相似度的去重

如果想去除高度相似但不完全相同的回复，使用相似度去重：

```bash
# 编辑距离方法（推荐）
python3 dedup_final.py training_data.jsonl output.jsonl \
  --key-field Response \
  --similarity \
  --method edit \
  --threshold 0.9

# Jaccard 相似度方法
python3 dedup_final.py training_data.jsonl output.jsonl \
  --key-field Response \
  --similarity \
  --method jaccard \
  --threshold 0.8
```

### 场景3：按问题去重

如果想去除相同问题的重复数据：

```bash
python3 dedup_final.py training_data.jsonl output.jsonl \
  --key-field input \
  --similarity \
  --threshold 0.95
```

### 场景4：支持嵌套字段

如果 Response 存储在 target 对象中：

```bash
python3 dedup_final.py training_data.jsonl output.jsonl \
  --key-field target.Response \
  --similarity
```

## 相似度计算方法

### 1. 编辑距离（Edit Distance）

**原理**：计算两个字符串转换所需的最少操作数（插入、删除、替换）

**优点**：
- 考虑字符顺序
- 对于部分相似的文本效果好
- 适合检测打字错误、小的修改

**缺点**：
- 计算复杂度较高（O(m*n)）
- 对于长文本较慢

**推荐阈值**：0.85-0.95

```
例子：
"我来帮您查一下"  vs  "我来帮您查查"
相似度 ≈ 0.71

"我来帮您查一下"  vs  "我来帮您查一下"
相似度 = 1.0
```

### 2. Jaccard 相似度

**原理**：基于字符集的交集和并集比例

**优点**：
- 计算速度快
- 对于字符集相似的文本效果好

**缺点**：
- 不考虑字符顺序
- 对于字符顺序变化敏感较低

**推荐阈值**：0.7-0.85

```
例子：
"我来帮您查一下"  vs  "我来帮您查一下"
集合1: {我,来,帮,您,查,一,下}
集合2: {我,来,帮,您,查,一,下}
相似度 = 1.0

"我来帮您查一下"  vs  "我来帮您查查"
集合1: {我,来,帮,您,查,一,下}
集合2: {我,来,帮,您,查}
相似度 = 5/7 ≈ 0.71
```

### 3. 余弦相似度（Cosine Similarity）

**原理**：基于字符频率的向量余弦相似度

**优点**：
- 考虑字符频率
- 对于频繁出现的字符更敏感
- 计算速度中等

**缺点**：
- 对于短文本可能不准确

**推荐阈值**：0.8-0.9

```
例子：
"我来帮您查一下"  vs  "我来帮您查一下"
相似度 = 1.0

"谢谢谢谢谢"  vs  "谢谢"
字符频率差异大
相似度 ≈ 0.6
```

## 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `input_file` | - | 输入文件路径（必需） |
| `output_file` | - | 输出文件路径（必需） |
| `--key-field` | Response | 用于去重的字段名 |
| `--similarity` | - | 使用相似度去重（否则完全相同） |
| `--method` | edit | 相似度计算方法（edit/jaccard/cosine） |
| `--threshold` | 0.8 | 相似度阈值（0-1） |
| `--top-n` | - | 完全相同时保留前 N 条 |

## 阈值选择建议

| 场景 | 方法 | 推荐阈值 | 说明 |
|------|------|---------|------|
| 完全去重 | - | 1.0 | 只移除完全相同的数据 |
| 严格去重 | edit | 0.95 | 移除几乎完全相同的数据 |
| 一般去重 | edit | 0.85 | 移除相似度较高的数据 |
| 宽松去重 | edit | 0.75 | 只移除高度相似的数据 |
| Jaccard | - | 0.8 | Jaccard 推荐阈值 |
| Cosine | - | 0.85 | 余弦相似度推荐阈值 |

## 性能考虑

### 时间复杂度

| 方法 | 复杂度 | 说明 |
|------|--------|------|
| 完全相同 | O(n) | 最快 |
| Jaccard | O(n²) | 中等 |
| 编辑距离 | O(n² × m²) | 最慢（m 为字符串长度） |
| 余弦相似度 | O(n² × m) | 中等偏慢 |

### 优化建议

对于大规模数据（100w+ 条）：

1. **使用 Jaccard 方法**（速度快）
2. **提高阈值**（减少比较次数）
3. **先完全去重**，再进行相似度去重
4. **按字段预分组**，只在组内比较

## 常见问题

### Q1: 应该选择哪个相似度方法？

**A**：
- 如果关心字符顺序 → 使用 **edit**（编辑距离）
- 如果只关心字符集 → 使用 **jaccard**
- 如果关心字符频率 → 使用 **cosine**

一般推荐 **edit**（编辑距离），效果最好。

### Q2: 阈值应该设置多高？

**A**：
- 0.95+：非常严格，只移除几乎完全相同的
- 0.85-0.95：一般推荐，移除明显相似的
- 0.7-0.85：宽松，会移除一些相似的
- <0.7：非常宽松，可能误删不同的数据

建议从 0.9 开始试验，根据结果调整。

### Q3: 完全相同和相似度去重有什么区别？

**A**：
- **完全相同**：只有完全相同的字符串才会被去重（相似度=1.0）
- **相似度去重**：相似度达到阈值的字符串都会被去重

例如：
```
"我来帮您查一下"
"我来帮您查查"

完全相同去重：不会被去重（不同）
相似度 0.9 去重：会被去重（相似度 ≈ 0.71 < 0.9，不会被去重）
相似度 0.7 去重：会被去重（相似度 ≈ 0.71 >= 0.7）
```

### Q4: 可以同时按多个字段去重吗？

**A**：暂不支持。如果需要，可以：
1. 先按字段1去重
2. 再按字段2去重
3. 或修改脚本支持多字段

## 数据格式要求

输入文件必须是 JSONL 格式，每行一个完整的 JSON 对象：

```json
{"input": "用户问题", "target": "{\"Response\": \"模型回复\"}", "session_id": "xxx"}
{"input": "另一个问题", "target": "{\"Response\": \"另一个回复\"}", "session_id": "yyy"}
```

## 与 dedup_and_quality.py 的区别

| 特性 | dedup_and_quality.py | dedup_final.py |
|------|-------------------|----------------|
| 使用时机 | 标注前（处理中间数据） | 标注后（处理最终数据） |
| 去重方法 | 完全相同 + 优选 | 完全相同 + 相似度 |
| 质量检查 | 有（与历史对话重复等） | 无 |
| 相似度算法 | 无 | 有（3种） |
| 字段灵活性 | 固定（input+Response） | 灵活（可指定任意字段） |

## 注意事项

1. **去重是不可逆的**：建议备份原始数据
2. **相似度计算较慢**：大规模数据可能需要较长时间
3. **阈值很重要**：不同阈值会导致完全不同的结果
4. **字段必须存在**：如果指定的字段不存在，会被当作空字符串处理

## 输出文件

输出是 JSONL 格式，与输入格式相同，只是去除了重复数据。
