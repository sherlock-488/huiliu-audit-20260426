# 回流数据过滤规则 V2 - 详细说明

## 📊 总览

**总规则数**: 9 个
**处理阶段**: 2 个（数据转换 + 数据过滤）

---

## 🔄 两阶段处理流程

### 阶段1: 数据转换 (txt → jsonl with user_reply)

**输入格式** (txt, TSV):
```
session_id	model_name	input_prompt	output
xxx	model_a	User: ...	{"Response": "...", "ResponseSolution": "..."}
```

**输出格式** (jsonl):
```json
{
  "session_id": "xxx",
  "model_name": "model_a",
  "user_reply": "用户：好的，谢谢",
  "input": "User: ...",
  "target": "{\"Response\": \"...\", \"ResponseSolution\": \"...\"}"
}
```

**关键操作**:
1. 从 `input_prompt` 中提取对话历史
2. 匹配客服回复后的用户回复
3. 保留完整的上下文信息

---

### 阶段2: 数据过滤

应用 9 个过滤规则，生成训练数据。

---

## 🎯 过滤规则详解

### 规则 1: 用户认可度检查 ⭐ (第一步，必须通过)

**检查对象**: `user_reply`（用户的最后一句回复）

**函数**: `check_user_approval(user_reply)`

**逻辑**: 
- ✅ 包含认可关键词 → 保留
- ❌ 不包含认可关键词 → 过滤

**认可关键词**:
```python
["好的", "好吧", "那行吧", "谢谢", "好的谢谢", "好", "行", "可以", "嗯嗯", "ok", "OK"]
```

**过滤原因**: `用户未认可`

**示例**:
```python
# ✅ 通过
user_reply = "用户：好的，谢谢"  # 包含"好的"和"谢谢"

# ❌ 过滤
user_reply = "用户：我知道了"  # 不包含任何认可关键词
```

---

### 规则 2: 用户负面反馈检查

**检查对象**: `user_reply`

**函数**: `check_user_negative(user_reply)`

**负面关键词**:
```python
["不行", "不可以", "不要", "不满意", "投诉", "退款", "差评", "垃圾", "骗人"]
```

**过滤原因**: `用户负面反馈`

**示例**:
```python
# ❌ 过滤
user_reply = "用户：不行，我不满意"  # 包含"不行"和"不满意"
```

---

### 规则 3: 非法工具/方案检查

**检查对象**: `target["ResponseSolution"]`, `target["DialogueAgreeSolution"]`

**函数**: `check_illegal_solution(response_solution, dialogue_agree_solution)`

**检查内容**:
1. 优惠券档位: 必须是 3/5/8/10/15/20 元
2. 运费券: 必须是 3 元
3. 退款比例: 必须是 0-100%
4. 工具名称: 必须在合法工具列表中

**合法工具列表**:
```python
[
    "加急调度", "系统催单", "智能跟单", "取消订单",
    "操作赔付-优惠券-x元", "操作赔付-运费券-3元",
    "选择商品", "小象单商品退款-xx%", "小象整单申请退款",
    "催促退款审核", "退审跟单", "引导自助修改订单", "引导自助联系骑手",
    "外呼骑手-修改订单", "外呼骑手-协商调换货品", "外呼骑手-催促配送",
    "外呼骑手-核实情况", "再次外呼骑手-……",
    "外呼门店-修改订单", "再次外呼门店-修改订单",
    "投诉骑手", "开发票", "会员卡退款", "管理会员自动续费"
]
```

**过滤原因**: `非法工具/方案`

**示例**:
```python
# ❌ 过滤
response_solution = "操作赔付-优惠券-7元"  # 7元不在合法档位

# ✅ 通过
response_solution = "操作赔付-优惠券-5元"  # 5元合法
```

---

### 规则 4: 过度承诺检查

**检查对象**: `target["Response"]`

**函数**: `check_overpromise(response)`

**过度承诺关键词**:
```python
["一定", "保证", "必须", "肯定", "绝对", "100%"]
```

**过滤原因**: `过度承诺`

**示例**:
```python
# ❌ 过滤
response = "我保证一定会帮您解决问题"  # 包含"保证"和"一定"

# ✅ 通过
response = "我会尽快帮您处理"
```

---

### 规则 5: 过多口语词检查

**检查对象**: `target["Response"]`

**函数**: `check_excessive_modal_particles(response, max_count=5)`

**口语词列表**:
```python
["哦", "啊", "呃", "嗯", "哈", "呢", "吧", "啦", "哟"]
```

**阈值**: 最多 5 个

**过滤原因**: `过多口语词`

**示例**:
```python
# ❌ 过滤（6个口语词）
response = "好的呢，我知道啦，嗯嗯，没问题哦，放心吧，谢谢哈"

# ✅ 通过（3个口语词）
response = "好的呢，我会帮您处理的哦"
```

---

### 规则 6: 输出长度检查

**检查对象**: `target["Response"]`

**函数**: `check_output_length(response, min_length=10, max_length=2000)`

**长度范围**: 10 - 2000 字符

**过滤原因**: `输出长度不合理`

**示例**:
```python
# ❌ 过滤
response = "好的"  # 只有2个字符，小于10

# ✅ 通过
response = "好的，我已经为您申请了退款"  # 14个字符
```

---

### 规则 7: 重复回复检查

**检查对象**: `target["Response"]`

**函数**: `check_repetitive_response(response)`

**检测逻辑**: 相同的3字符片段重复超过3次

**过滤原因**: `重复回复`

**示例**:
```python
# ❌ 过滤
response = "好的好的好的好的"  # "好的"重复4次

# ✅ 通过
response = "好的，我会帮您处理的"
```

---

### 规则 8: 退货/取回关键词检查 🆕

**检查对象**: `target["Response"]`

**函数**: `check_return_keywords(response)`

**说明**: 智能客服没有退货取回能力，需要过滤相关回复

**退货/取回关键词**:
```python
[
    '退货', '退回', '取回', '保留商品', '上门取件', '取件',
    '自行处理', '丢弃', '扔掉', '寄回', '退还',
    '商品处理', '物品处理', '货物退回',
]
```

**过滤原因**: `包含退货/取回关键词`

**示例**:
```python
# ❌ 过滤
response = "您可以安排上门取件，我们会取回商品"

# ✅ 通过
response = "我已经为您申请了退款"
```

---

### 规则 9: 其他业务名称检查 🆕

**检查对象**: `target["Response"]`

**函数**: `check_other_business_keywords(response)`

**说明**: 避免混淆其他业务线

**其他业务名称**:
```python
[
    '拼好饭', '放心吃', '美团买菜', '美团优选', '美团闪购',
    '象大厨', '象会员', '象超市',
]
```

**过滤原因**: `包含其他业务名称`

**示例**:
```python
# ❌ 过滤
response = "建议您试试拼好饭，那边有更多优惠"

# ✅ 通过
response = "我会为您申请优惠券"
```

## 📋 完整过滤流程

```
输入数据（带 user_reply）
  ↓
1. 用户认可度检查 (user_reply) ⭐ 必须通过
  ↓
2. 用户负面反馈检查 (user_reply)
  ↓
3. 非法工具/方案检查 (ResponseSolution, DialogueAgreeSolution)
  ↓
4. 过度承诺检查 (Response)
  ↓
5. 过多口语词检查 (Response)
  ↓
6. 输出长度检查 (Response)
  ↓
7. 重复回复检查 (Response)
  ↓
8. 退货/取回关键词检查 (Response) 🆕
  ↓
9. 其他业务名称检查 (Response) 🆕
  ↓
通过所有检查 → 输出训练数据
```

---

## 🎯 关键改进点

### V1 → V2 主要变化

1. **新增阶段1**: 数据转换，提取 `user_reply`
2. **调整检查顺序**: `user_reply` 相关检查放在最前面
3. **明确检查对象**: 
   - `user_reply` → 用户认可度、负面反馈
   - `Response` → 过度承诺、口语词、长度、重复、退货、业务名称
   - `ResponseSolution` → 非法工具/方案
4. **第一步必须通过**: 用户认可度是最基础的条件
