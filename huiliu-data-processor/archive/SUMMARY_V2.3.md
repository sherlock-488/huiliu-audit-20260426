## 🔧 规则实现细节

### 规则 15: 说要给用户打电话

```python
def check_call_user(response):
    # 规则1: 我...给/为你/您打/拨/回拨/来电/致电/去电/回电
    if re.search(r"我.{0,6}(给|为)[你您](打|拨|回拨|来电|致电|去电|回电)", response):
        return True

    # 规则2: 我...联系/外呼/致电/去电你/您
    if re.search(r"我.{0,4}(联系|外呼|致电|去电)[你您]", response):
        return True

    # 规则3: 给你/您回电/来电
    if re.search(r"(给|为)[你您](回电|来电)", response):
        return True

    # 规则4: 外呼 + 注意接听
    if re.search(r"外呼", response) and re.search(r"注意接听", response):
        return True

    return False
```

**匹配示例**:
- ❌ "我马上给您打电话核实情况"
- ❌ "我为您联系骑手，稍后会给您回电"
- ❌ "我这边会致电您了解详细情况"
- ❌ "我会外呼骑手核实情况，请您注意接听电话"

## 🎓 技术亮点

### 1. 两阶段处理架构
- **阶段 1**: 数据转换 - 从对话历史中提取 `user_reply`
- **阶段 2**: 数据过滤 - 应用 16 个规则

### 2. 智能参数传递
- `user_reply`: 用户认可度、用户负面反馈、转人工关键词
- `Response`: 过度承诺、口语词、长度、重复、退货、业务名称、打电话、升级转接
- `ResponseSolution`: 非法工具、信号逻辑、赔付冲突
- `input_prompt`: 信号提取、赔付权限、退款豁免
- `context`: 对话上下文（客服声称真人）

### 3. 正则表达式优化
- 使用负向预查 `(?!申请)` 排除过程态表述
- 使用可选匹配 `.{0,6}` 处理中间插入词
- 使用多条件组合提高匹配准确率

### 4. 豁免条件处理
- 退款越权规则：商品状态为"已全部退款"时豁免
- 信号逻辑规则：特定场景下的信号匹配

### 5. 测试驱动开发
- 每个新规则都有专门的测试文件
- 测试覆盖率 100%
- 测试用例包含正例和负例
