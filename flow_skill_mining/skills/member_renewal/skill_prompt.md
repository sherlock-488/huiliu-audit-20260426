# 会员场景-续费问题 流程 skill 草案

适用场景：仅用于 `会员场景-续费问题`。本 skill 只约束流程动作，不负责生成漂亮话术。

必读信号：天气, 是否咨询高峰期, 用户分层标签, 是否会员, 用户和门店的通话记录, 用户和骑手的通话记录, 卡数量, 订单状态, 订单类型, 配送类型, 商品数量, 订单来源, 是否为配送单, 是否为自提单, 是否外卖单, 门店名称, 城市名称, 订单是否含追加单, 订单是否含加工品, 支付类型

ResponseSolution 选择规则：优先根据用户当前诉求、订单/退款/赔付/商品选择状态选择可执行方案；高频候选方案包括：管理会员自动续费, 会员卡退款。如果当前只是询问状态、缺商品、缺图片、缺退款原因，ResponseSolution 可以为“无”，但必须有明确 no_action_reason。

DialogueAgreeSolution 填写规则：只有用户已经明确同意取消、退款、赔付、会员卡退款等待确认方案时才填写。用户未明确同意时，即使客服本轮提出方案，DialogueAgreeSolution 也应为“无”。疑似需要确认的方案：会员卡退款。

禁止项：不要把 DialogueAgreeSolution 简单复制 ResponseSolution；不可赔付时不得填赔付方案；不可取消时不得填取消订单；需要商品或图片前置时不得直接进入单商品退款。

ResponseSolution=无 的合理原因：澄清诉求、请用户选择商品、请用户上传图片、解释订单/退款状态、等待工具结果、动作后的追问或结束会话。
